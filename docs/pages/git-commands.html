Справочник команд Git

Основные команды, которые использовались в лабораторной работе

Настройка

git config --global user.name "Имя" Настройка имени пользователя
git config --global user.email "email" Настройка email пользователя
git config --list Просмотр всех настроек

Создание репозитория

git init Инициализация нового репозитория
git clone URL Клонирование удаленного репозитория
git status Проверка состояния репозитория

Работа с изменениями

git add . Добавить все файлы
git add имя_файла Добавить конкретный файл
git commit -m "Сообщение" Создать коммит
git log --oneline Просмотр истории коммитов

Ветвление

git branch Показать все ветки
git branch имя_ветки Создать новую ветку
git checkout имя_ветки Переключиться на ветку
git merge имя_ветки Слить ветку с текущей

Работа с GitHub

git remote add origin URL Добавить удаленный репозиторий
git push -u origin main Отправить изменения на GitHub
git pull Получить изменения с GitHub
git remote -v Показать подключенные репозитории

Отмена изменений

git restore --staged файл Отменить добавление файла
git reset HEAD~1 Отменить последний коммит
git checkout -- файл Отменить изменения в файле

Полезные советы

Всегда пишите понятные сообщения коммитов

Вместо "fix" пишите "Исправлена ошибка в функции расчета"

Часто делайте коммиты

Каждый коммит должен содержать логически завершенное изменение

Проверяйте статус перед коммитом

Всегда используйте git status перед добавлением файлов
